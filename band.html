Garageband Hero

Watch Gameplay Source Code

I've always loved the concept of party-driven RPGs. You meet friends along a journey, and as a team do what you couldn't do alone. This demo is more of a proof of concept, than a game, as I believe this will take more than a one-man-team to complete. It is definitely the most ambitious idea I’ve had thus far. All that said, I am very passionate about progressing with this, as I think it is a unique combination of genres and I want to see more creative ways of using the RPG battle systems. I have challenged myself to create the base systems that can be developed further.

The game currently features an interactable town where the player can walk around in. If the player enters the café, a battle starts. The player’s band challenges a rival band with a crowd onlooking. Instead of a health system, the crowd is swayed by the talent of each band. When the crowd is swung far enough in one direction, the battle ends.

State Machine Battle System

The turn-based battling system was created using a State Machine. This means that a game manager controls what phase of battle the game is in currently, and can transition to the next state based on conditions. The game manager alters behavior and player controls based on what phase is currently active. Each state has a transition to a following state, or the end state once the battle is complete.
State machines remove the need for each module to have long conditional chains in order to perform certain methods. Instead, the game manager can call specific methods for each module, ignoring functionality that does not pertain to the current state.

Strategy and Movesets

In the planning phase, the player must select moves for each band member, and only when all band members are ready can the game progress to the performance phase. The idea is to select moves that get the band the most Star Points and swing the crowd to their side.

Crowd Generation and Responsiveness

Since the crowd determines the winner of the battle, it was important to have a crowd system that felt alive. To implement this, I created a modular prefab that I can assign skin and shirt colors. I created a few crowd dancing animations, and then developed a system of generating a grid of these prefabs. The entire crowd is controlled by one script, calling each of the crowd objects when an event occurs. Currently the crowd can go from idle to dancing, and turn around to face the band that is currently active in battle.
